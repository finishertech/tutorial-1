13. Removendo código que não utilizaremos neste tutorial

Execute esta linha de comando:

3$ rm -f README.md public/*.{png,json} src/{logo.svg,serviceWorker.js,*[Tt]est*.js}
Neste tutorial, apesar de eu trabalhar bastante com TDD em meus projetos, eu decidi que não falarei sobre testes, agora. Dessa forma, os códigos de teste estão sendo removidos com o comando acima. Acompanhe o lançamento de próximos tutoriais onde falarei sobre esse assunto.

A execução desse comando causará um Failed to compile. Isso poderá ser visto na tela que apresenta a saída do comando yarn start (shell 2). Vamos prosseguir, mesmo assim, pois corrigiremos isso logo a frente.

Veja o status do git e adicione todas essas mudanças a sua staging area:

3$ git status
3$ git add -A
3$ !-2
Mania: usar o histórico de comandos do Bash

Eu sou muito preguiçoso para escrever linhas de comando. Por isso, adoro as possibilidades que o Bash nos oferece para reexecutar um comando que já foi pro seu histórico. Sendo assim, prefiro digitar um "!!" para o Bash reexecutar o último comando que escrevi e, da mesma forma, "!-2" para o penúltimo.
